## TDesign 组件路径错误 - 已解决 ✅

您遇到的错误：
```
app.json: ["usingComponents"]["t-toast"]: "tdesign-miniprogram/toast/toast"
未找到组件
```

**这个问题已经在最新代码中修复！**

---

## 📥 获取修复

### 方法1：拉取最新代码（如果使用 Git）

```bash
git pull origin main
# 或
git pull origin copilot/implement-wechat-mini-program-mvp
```

### 方法2：手动修复（如果没有使用 Git）

编辑 `app.json` 文件，删除 `usingComponents` 部分：

**删除这几行：**
```json
"usingComponents": {
  "t-toast": "tdesign-miniprogram/toast/toast",
  "t-message": "tdesign-miniprogram/message/message"
},
```

**完整的 app.json 应该是这样（第 50-53 行）：**
```json
  },
  "style": "v2",
  "sitemapLocation": "sitemap.json",
  "lazyCodeLoading": "requiredComponents"
}
```

---

## ❓ 为什么会出现这个错误？

**原因：**
- Toast 和 Message 组件不应该全局注册
- 这些组件通过 API 调用（如 `wx.showToast()`）
- 全局注册会影响性能

**修复方式：**
- 移除了全局组件配置
- 各页面按需引用组件
- 使用原生 API 显示提示信息
